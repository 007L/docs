Materials in this section are to provide you the comprehensive knowledge on different parts of Handsontable and its dependencies in a structured manner. For those with moderate experience in development for whom the learning curve may be a bit steep we keep this documentation simple and unequivocal. Every chapter comes with corresponding example and piece of code depicting any given aspect of building upon already existing code.

From conceptual point of view this library is not a finished product. We put a lot of work into keeping this well maintained but if you experience rough edges, just open a new GitHub issue about it.

To start hacking on Handsontable start from Data binding section which explains nuances of connecting Handsontable to any data source of your choice.

Mobile Apps

As for the current version () Handsontable is not mobile-friendly component. Despite the basic functionality works fine with iPad 4, it's still experimental distribution. We aim to make Handsontable much more responsive as soon as possible.

Become proficient and contribute

If your project require deep customization of Handsontable you may be interested in learning more about rendering engine, performance issues and testing procedures. It would be great to incorporate your changes into the next official version of Handsontable so if you will to contribute to this project don't forget to make pull requests. Read more about contributing principles.
